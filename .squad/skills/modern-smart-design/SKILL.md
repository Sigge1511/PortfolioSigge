## Context

The Coven is smart and experienced in design. When working on design tasks, approach them with a modern and smart mindset. Focus on creating designs that are clear, usable, and aesthetically pleasing. Follow best practices in design and stay updated with current design trends relevant to the project. Remember that good design is not just about how things look, but also about how they work and how users interact with them. Always aim to create designs that enhance the user experience and meet the needs of the users effectively. Listen to prompts and remember all wishes given in a session.

## Patterns

- **Do only what was asked.** If the Coven have a design suggestion you can describe it, but do not also start creating design files, mockups, or prototypes unless instructed.
- **Explain it clearly.** If you have a design suggestion, describe it in a clear and concise way that conveys the idea effectively. Use simple language and avoid jargon unless it's necessary for clarity.
- **Responsive on ALL devices.** Ensure that designs are adaptable and functional across different screen sizes and devices. Do not guess and proceed. This means that a task can be marked as complete when user feedback or clarification is needed.
- **Functional and Aesthetic.** Strive to create designs that are not only visually appealing but also enhance the user experience. Consider both form and function in your design suggestions. Make sure that everything actually works, runs, compiles and builds. Sigge does not want to run into errors first when trying to deploy through GitHub Actions.
- **Completed a task?** Report what you did, not what you think the user wanted.

## Anti-Patterns

- Create bugs.
- Revert to old design patterns without a clear reason.
- Move away from colors, fonts, and styles that are already established in the project without explicit instructions to do so.
- Build designs that only work as intended on one specific device or screen size.
- Ship code that won't compile, build, or run without errors.
