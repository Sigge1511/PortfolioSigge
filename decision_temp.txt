# Decision: Convert to Pure React/Vite (Remove ASP.NET)

**Date:** 2025-07-15
**Author:** Selene (Frontend Engineer)
**Status:** Implemented

## Context

The project was scaffolded as an ASP.NET Core + React hybrid. The React frontend lived in `portfoliosigge.client/` and was tightly coupled to the .NET backend via:
- `vite.config.ts` generating dotnet dev-certs via `child_process.spawnSync`
- ASPNETCORE environment variables for port and HTTPS configuration
- A proxy to `/weatherforecast` (an ASP.NET demo endpoint)
- HTTPS server config reading generated PEM files

The portfolio is a static site -- no backend is needed.

## Decision

Convert the project to a standalone React/Vite SPA:
1. Move React source to the repo root (from `portfoliosigge.client/`)
2. Replace the complex vite.config.ts with a minimal config (plugin + @ alias only)
3. Delete `portfoliosigge.client/` directory
4. Rename package from `portfoliosigge.client` to `portfoliosigge`

## What Was Removed

- `dotnet dev-certs` HTTPS certificate generation on startup
- ASPNETCORE_HTTPS_PORT / ASPNETCORE_URLS environment variable coupling
- `/weatherforecast` proxy rule
- HTTPS server configuration (key/cert file reads)
- The `portfoliosigge.client/` subdirectory

## What Remains

- React 19 + TypeScript + Vite
- `@vitejs/plugin-react`
- @ alias for ./src
- All existing `src/` components and styles

## Rationale

A pure static React/Vite setup is simpler, has no external toolchain dependencies (no .NET SDK required to run `npm run dev`), and is directly deployable to Strato static hosting.
